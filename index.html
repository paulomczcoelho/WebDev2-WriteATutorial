UNDERSTANDING THE DOM (DOCUMENT OBJECT MODEL)

The DOM explained in an easy way.

A lot of HTML was studied, the first tags, I learned CSS, made beautiful forms, amazing buttons, responsive pages and started showing everyone how amazing that is, but there comes a time when you want to take a step further in your learning and ask yourself : ”But how can I give movement to my page? I so wanted that button, for example, to change something about her!”. That's where the DOM comes in, something you constantly hear about, but don't know how to explain exactly what it is. You know those cool animations that you see around and you say “Wow, I wanted to do something like this so much, but how?”, they are made by manipulating the DOM (Document Object Model), and this article will explain from scratch what it is and how to manipulate it.

So what is the DOM?

The DOM (Document Object Model) is an interface that represents how HTML and XML documents are read by your browser. After the browser reads your HTML document, it creates an object that makes a structured representation of your document and defines ways in which this structure can be accessed. We can access and manipulate the DOM with JavaScript, it's the easiest and most used way.

What is the advantages of DOM?

With it you have endless possibilities, you can create applications that update page data without requiring an update. You can also create applications that are customizable by the user, change the page layout without needing to update. Drag, move, delete elements. That is, you have infinite possibilities, thousands of things you can do by manipulating the DOM, just use your creativity.

In this image we see the structure of the DOM, its tags and how it is assembled by the browser. On this basis, we have 4 important points that you will see a lot from now on: **Document: **which as the name says, takes care of HTML documents. **Elements: **are all the tags that are in HTML or XML files that are transformed into elements of the DOM tree. Methods

The DOM has many methods, they are the ones that make the connection between the nodes (elements) and the events, something I will talk about below. I will show the most used methods and what they are for too. Remembering that there are several, for all types and you can see them all here.

getElementById()

This method returns the element containing the name of the passed ID. As the IDs must be unique, it is a very useful method to get only the desired element.

getElementsByClassName()

This method returns an HTMLCollection of all elements that contain the name of the passed class.

myContainer: specific element that matches the passed selector.
.container: selector passed, if there was none it would return null.

getElementsByTagName()

In the same way as the above method, it also returns an HTMLCollection but with one difference: this method returns all elements containing the passed name tag.

buttons: specific element that matches the passed selector.
button: tag name passed.

querySelector()

Returns the first element that matches the passed CSS selector. Remembering that the selector must follow the CSS syntax. If it has no selector, it will return _null. _

resetButton: first element that matches the passed selector.
#reset: selector passed, if there was none it would return null.

querySelectorAll()

Very similar to querySelector(), but it has only one difference: it returns all elements that match the passed CSS selector. The selector must also follow the CSS syntax, if there is none it will return null.

myButtons: elements that match the passed selector.
#buttons: selector passed, as in this case there is no selector with that name, it returns null.
These are just 3 methods of DOM, there are several and some are quite used, for example createElement() which creates a new HTML element using the name of the tag to be created, the setAttribute() that you can set new attributes for HTML elements and many others.
